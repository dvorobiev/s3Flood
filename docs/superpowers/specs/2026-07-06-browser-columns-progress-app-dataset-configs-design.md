# Дизайн: колонки и прогресс в браузере, датасет уровня приложения, упрощение имён конфигов

Дата: 2026-07-06
Ветка: feature/bucket-browser (продолжение, v0.12.0-beta)

## Мотивация

Фидбек пользователя после ручного теста браузера v2:
1. Колонки панелей не подписаны и не отделены — нужен вид как в Norton/Midnight Commander.
2. Файловые операции показывают прогресс только строкой статуса — нужно модальное окно прогресса.
3. Датасет должен задаваться на уровне приложения, а не в каждом конфиге прогона.
4. Имена конфигов не должны требовать префикса `config.` — валидность определяется содержимым файла.

## 1. Браузер: колонки в стиле Norton Commander

Изменения в `render_panel_lines` (`browser.py`):

- Под титулом панели добавляется строка заголовков колонок.
- В каждой строке — вертикальные разделители `│` между колонками.

```
┤ /Users/dvorobiev/data ├─────────────
 Имя                 │  Размер │ Дата
 ..                  │         │
 medium_0042.bin     │ 24.5 MB │ 2026-07-03 14:02
```

- Локальная панель и режим `list`: `Имя │ Размер │ Дата` (бейдж ⊙N остаётся в колонке даты).
- Режим `versions`: `Версия │ Размер │ Дата / метки`.
- Режим `buckets`: `Бакет │ пометка` (колонка размера не показывается).
- Ширины: размер — 9 символов, дата/метки — фиксированная, имя — остаток (минимум 12).
- Заголовок колонок имеет собственный стиль (`class:panel.columns`, приглушённый).

## 2. Браузер: модальное окно прогресса

- Новый dataclass `ProgressState`: название операции, стрелка направления, имя текущего
  файла, счётчик n/m, байты обработано/всего, число ошибок, флаг отмены.
- UI: `Float` поверх панелей через `FloatContainer` + `ConditionalContainer`,
  показывается пока `app.progress is not None`. Внутри: заголовок операции, текущий
  файл, прогресс-бар по числу файлов, прогресс-бар по объёму, счётчик ошибок,
  подсказка «Esc — отмена».
- Все пакетные операции (`_op_transfer_batch`, `_op_delete_batch`,
  `_op_download_version`, `_op_restore`) обновляют `ProgressState` вместо
  `status_msg`. Прогресс по объёму — сумма размеров уже завершённых файлов
  (вывод aws cli не парсим).
- На время операции обычные клавиши блокируются фильтром `no_progress`;
  Esc ставит флаг отмены — текущий файл дозавершается, остальные не начинаются.
- По завершении окно закрывается, итог выводится в статус-бар (как сейчас).

## 3. Датасет на уровне приложения

- Новый модуль `app_settings.py`: `load_app_settings()` / `save_app_settings()`
  для файла `.s3flood.yml` в рабочей папке. Ключ: `dataset_dir` (структура —
  плоский YAML-маппинг, задел под будущие настройки).
- Мастер «Создать датасет» (`create_dataset_menu`) после успешной генерации
  записывает путь в `.s3flood.yml`.
- `resolve_run_settings` (`config.py`): приоритет источников `data_dir`:
  CLI `--data-dir` → `.s3flood.yml` → дефолт `./data`.
  Поле `data_dir` из конфига прогона игнорируется; если оно присутствует —
  печатается предупреждение о том, что датасет теперь задаётся приложением.
- Редактор конфигов и визард создания больше не показывают поле `data_dir`;
  `config.sample.yaml` обновляется (упоминание `.s3flood.yml`).
- Браузер бакета стартует из датасета приложения (та же цепочка приоритетов).

## 4. Конфиги: имена без префикса, валидность по содержимому

- Новая функция `discover_configs(cwd: Path) -> list[Path]` в `config.py`:
  glob `*.yml` + `*.yaml`, файл считается конфигом прогона, если парсится в
  YAML-маппинг и содержит секцию `run` (маппинг) либо хотя бы один из ключей:
  `endpoint`, `endpoints`, `bucket`, `profile`.
- Заменяет все 4 места с glob `config*.yml|yaml` в `interactive.py`
  (запуск теста, редактирование, проверка, браузер).
- Старые `config.*.yaml` остаются валидными (по содержимому); новые файлы
  можно называть свободно: `kazan.yml`, `tape.yml`.
- Дефолтное имя в визарде создания — `new.yml`.
- `.s3flood.yml` — dotfile, в glob не попадает и в списках не показывается.

## Тестирование

- Рендер: строка заголовков и разделители во всех трёх режимах панели.
- ProgressState: обновление счётчиков, суммирование байт, отмена по флагу
  (оставшиеся файлы не обрабатываются), ошибки учитываются.
- `discover_configs`: валидные конфиги (с `run` и плоские), мусорные yml
  (список, скаляр, не-yaml), пустые файлы, `.s3flood.yml` исключён.
- Приоритет data_dir: CLI > app settings > дефолт; конфиг с data_dir —
  предупреждение и игнор.
- Мастер датасета пишет `dataset_dir` в `.s3flood.yml`.

## Вне объёма

- Байтовый прогресс внутри одного файла (парсинг вывода aws cli).
- Глобальные настройки `~/.s3flood/` — только локальный файл в cwd.
- Миграция/переименование существующих конфигов.
