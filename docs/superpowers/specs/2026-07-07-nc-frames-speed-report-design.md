# Дизайн: рамки NC вокруг панелей браузера, итоговый отчёт с упором на скорость

Дата: 2026-07-07
Ветка: feature/nc-frames-speed-report (после релиза 0.12.0-beta)

## Мотивация

Фидбек пользователя со скриншотом Norton Commander:
1. Панели браузера должны выглядеть как в NC: рамки вокруг панелей, титул в верхней
   рамке, подписанные колонки, сводная строка внизу панели («N bytes in M files»).
   Синий фон не обязателен.
2. Итоговый отчёт после прогона должен делать упор на скорость, а не на латентность.

## 1. Рамки NC в браузере

Каждая панель оборачивается в prompt_toolkit `Frame` (уже используется для окна
прогресса) — рамка ┌─┐│└┘ с титулом в верхней кромке:

```
┌ /Users/dvorobiev/data ──────────────┐
│ Имя              │  Размер │ Дата   │
│ ..               │         │        │
│ medium_0042.bin  │ 24.5 MB │ 07-03  │
│                                     │
│ 5 объектов · 124.5 MB               │
└─────────────────────────────────────┘
```

- Титул рисует Frame → из `render_panel_lines` титульная строка удаляется
  (первой строкой становится заголовок колонок); `_cursor_point` возвращается
  к `+1`.
- Фокус панели показывается стилем рамки/титула: у Frame динамический
  `style=lambda` — `class:frame.focused` (акцентный #00d7ff bold) против
  приглушённого `class:frame.dim`.
- Внизу каждой панели, внутри рамки — сводная строка (отдельное окно высотой 1):
  обычно «N объектов · суммарный размер»; при выделении Space —
  «выделено n · размер выделенного» (стиль row.marked, как жёлтая строка NC).
  Чистая функция `panel_summary(panel) -> str`.
- Компоновка: каждая сторона — `Frame(HSplit([окно строк, окно сводки]), title=...)`.
- Строка статуса и клавишная панель внизу экрана — без изменений.

## 2. Итоговый отчёт: упор на скорость

`print_summary` (executor.py) перестраивается — скорость первой и крупно:

1. **Панель «Скорость»** (rich Panel, сразу под заголовком):
   - сквозная скорость: `(write_bytes + read_bytes) / duration_sec` MB/s — крупно;
   - раздельно `↑ запись N MB/s` и `↓ чтение M MB/s` (сквозные, уже считаются);
   - операций в секунду: `(write_ok + read_ok) / duration_sec`;
   - пиковая скорость по секундам из `timeline` (MB/s лучшей секунды);
   - спарклайн MB/s по времени из `timeline` (reuse `sparkline()` из dashboard.py).
2. Таблица «Пропускная способность» остаётся, добавляется строка **Итого**
   (сумма операций, объёма, сквозная скорость).
3. Латентность — после скорости, компактнее: только p50 / p95 / p99 / avg
   (убираем p90 и max из таблицы; в report.json всё остаётся как было).
4. Ошибки и строка времени — без изменений.

Новые чистые функции в metrics.py (тестируемые):
- `timeline_speeds(timeline: list[dict]) -> list[float]` — MB/s на бакет
  (учитывает укрупнение шага: делит байты на реальную ширину бакета);
- `summary_speed_stats(summary: dict) -> dict` — total_MBps, write_MBps,
  read_MBps, ops_per_sec, peak_MBps (0.0 при пустых данных).

`report.json` НЕ меняется — только консольный вывод.

## Тестирование

- panel_summary: пусто / N объектов / есть выделение (счёт и байты только
  по выделенным); «..» и каталоги не входят в счёт объектов.
- render_panel_lines без титульной строки: первая строка — заголовки колонок.
- timeline_speeds: пустой timeline, один бакет, укрупнённый шаг.
- summary_speed_stats: обычный summary, нулевой duration, отсутствующий timeline.
- Существующие тесты рендера обновляются (сдвиг индексов строк).

## Вне объёма

- Синий фон и полная цветовая схема NC.
- Байтовые прогресс-бары внутри одного файла.
- Изменения report.json.
