# Дизайн: фиксированная ширина панелей, внешняя двойная рамка, акцент фокуса, умное сокращение имён

Дата: 2026-07-08
Ветка: feature/panel-width-double-frame (после релиза 0.12.1-beta)

## Мотивация

Фидбек пользователя после визуального теста браузера бакета (0.12.1-beta):
1. Панель бакета «схлопывается» по ширине при старте приложения и при выборе бакета —
   нужна жёсткая фиксация 50%/50% экрана на каждую панель.
2. Внешнюю рамку приложения можно сделать двойной, акценты — ярче.
3. Для длинных имён файлов, не помещающихся в панель, показывать умное сокращение
   «начало…конец.расширение» вместо простого обрезания хвоста.

## 1. Фиксированная ширина панелей 50/50

### Причина схлопывания

Обёртки панелей (`left_frame`/`right_frame`, `HSplit([Frame(...)], style=...)`) не имеют
явно заданной ширины. prompt_toolkit вычисляет предпочтительную ширину контейнера из
содержимого его окна на момент рендера. Пока панель в состоянии `loading` (текст
«⠋ загрузка…», короткая строка) или показывает короткий список бакетов, библиотека
фиксирует эту короткую ширину как предпочтительную для всего `HSplit`-контейнера — и
`VSplit` перестаёт делить пространство поровну, отдавая панели ровно столько, сколько
«попросил» её временный контент.

### Фикс

Явно задать `width=Dimension(weight=1)` (импорт `Dimension as D` из
`prompt_toolkit.layout.dimension`) на обёртках обеих панелей:

```python
left_frame = HSplit([...], width=D(weight=1), style=lambda: ...)
right_frame = HSplit([...], width=D(weight=1), style=lambda: ...)
```

С одинаковым весом `VSplit` всегда делит доступную ширину ровно пополам, независимо от
контента дочерних окон.

### Пересчёт `_panel_width()` с учётом внешней рамки

С добавлением внешней рамки (см. п.2) доступная ширина уменьшается на 2 колонки
(левая и правая граница внешней рамки). Новая формула:

```python
def _panel_width(self) -> int:
    cols = self._term_cols()
    inner_cols = cols - 2          # внешняя рамка
    panel_area = inner_cols // 2   # половина на панель (VSplit 50/50)
    return max(panel_area - 2, 20) # минус собственная рамка панели (Frame)
```

### Тестирование

Аналогично прошлой итерации (где похожая проблема с шириной уже ловилась только
реальным рендером): тест через настоящий prompt_toolkit `Screen`
(`write_to_screen` на `Application` с `DummyOutput` фиксированной ширины) —
проверяет, что при разных сценариях контента (loading vs список файлов vs список
бакетов) обе панели получают одинаковую ширину экрана, а не только то, что чистая
функция `_panel_width()` возвращает ожидаемое число.

## 2. Внешняя двойная рамка вокруг всего приложения

Одна общая рамка `╔═╗ ║ ╚═╝` вокруг всего приложения — панели, строка статуса и
клавишная панель внизу оказываются внутри неё целиком (как в референсном
скриншоте Norton Commander). Без текста-заголовка на рамке — просто контур.

Встроенный `prompt_toolkit.widgets.Frame` использует зашитые одинарные символы
рамки (`Border.HORIZONTAL/VERTICAL/...`) без возможности подмены на двойные без
глобального монки-патча (что задело бы и внутренние рамки панелей). Поэтому внешняя
рамка — три новых окна-примитива, а не `Frame`:

```python
def _outer_line(self, left: str, fill: str, right: str) -> list[tuple[str, str]]:
    cols = self._term_cols()
    return [("class:outer.border", left + fill * max(cols - 2, 0) + right)]

# верх: Window(height=1, content=FormattedTextControl(lambda: self._outer_line("╔", "═", "╗")))
# низ:  Window(height=1, content=FormattedTextControl(lambda: self._outer_line("╚", "═", "╝")))
# бока: VSplit([Window(width=1, char="║", style="class:outer.border"),
#               <текущий root: FloatContainer(HSplit([body, status, keybar]), floats=[...])>,
#               Window(width=1, char="║", style="class:outer.border")])
```

Итоговая раскладка приложения:

```
root = HSplit([
    Window(height=1, ...)  # верхняя двойная граница
    VSplit([Window(width=1, "║"), inner, Window(width=1, "║")])
    Window(height=1, ...)  # нижняя двойная граница
])
```

где `inner` — существующий `FloatContainer(content=HSplit([body, status, keybar]),
floats=[progress_float])` без изменений (модальное окно прогресса по-прежнему
центрируется относительно этой внутренней области).

Внутренние рамки панелей (`Frame` вокруг каждой из двух панелей) остаются
одинарными — меняется только внешний контур.

## 3. Акцент только на фокусе

Меняется исключительно стиль активной (сфокусированной) панели. Текущее:
`"panelfocus frame.border": "fg:#00d7ff bold"`. Новое — более яркий и насыщенный тон:
`"panelfocus frame.border": "fg:ansibrightcyan bold"`,
`"panelfocus frame.label": "fg:ansibrightcyan bold"`,
`"panelfocus panel.summary": "fg:ansibrightcyan bold"`.
Стили каталогов (`row.dir`), обычного выделения (`row.marked`) и неактивной панели —
без изменений.

## 4. Умное сокращение длинных имён (middle-ellipsis)

Новая чистая функция в `browser.py`:

```python
def truncate_middle(name: str, width: int) -> str:
    """Сокращает длинную строку до width символов: начало…конец.

    Хвост включает конец исходной строки — для имён файлов это естественно
    сохраняет расширение, т.к. оно всегда в конце.
    """
    if len(name) <= width:
        return name
    if width <= 1:
        return name[:width]
    avail = width - 1  # 1 символ на "…"
    tail_len = avail // 2
    head_len = avail - tail_len
    return name[:head_len] + "…" + (name[-tail_len:] if tail_len else "")
```

`format_columns` использует её перед выравниванием по ширине колонки:

```python
def format_columns(name: str, size: str, meta: str, width: int) -> str:
    name_w = max(width - 33, 12)
    display_name = truncate_middle(name, name_w)
    return f"{display_name:<{name_w}} │{size:>9} │ {meta}"
```

(раньше усечение и выравнивание делались одним format-спеком `{name:<{w}.{w}}`,
который просто обрезал хвост без учёта расширения — теперь усечение выделено в
отдельный шаг перед выравниванием.)

Работает во всех трёх режимах панели (список файлов/объектов, версии, бакеты),
так как все они рендерятся через общую `format_columns`.

## Тестирование (сводно)

- `_panel_width()` / реальный рендер: обе панели получают одинаковую ширину при
  loading, при списке файлов, при списке бакетов, при разных ширинах терминала.
- Внешняя рамка: верхняя/нижняя линии ровно `cols` символов, начинаются/заканчиваются
  правильными угловыми символами; боковые `║` присутствуют по всей высоте.
- Стиль фокуса: активная панель использует новый яркий класс, неактивная — старый.
- `truncate_middle`: короткое имя не трогается, длинное — `head…tail` ровно `width`
  символов, расширение оказывается в хвосте для типичных случаев (`.tar.gz`,
  `.bin`), край-кейсы (`width<=1`, пустая строка, имя без точки).

## Вне объёма

- Настраиваемая палитра/тема (одна фиксированная схема ярких акцентов).
- Заголовок текста на внешней рамке.
- Прокрутка длинных путей текущей директории в титуле панели.
